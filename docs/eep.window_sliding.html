A sliding window. Sliding windows (here) have a fixed a-priori known size. Example: Sliding window of size 2 computing sum of values.


    t0     t1      (emit)   t2             (emit)       tN
  +---+  +---+---+          -...-...-
  | 1 |  | 2 | 1 |   <3>    : x : x :
  +---+  +---+---+          _...+---+---+               ...
             | 2 |              | 2 | 3 |    <5>
             +---+              +---+---+
                                    | 4 |
                                    +---+

So, we only ever need N active windows to emit a result for every input event once all windows are open. Hence, use a ring buffer and preallocate the slots (functions).

This is the only window implementation that actually needs a queue (ring buffer) in this simple embedded event processing engine.
